# Task Management

## Purpose

This document defines how tasks should be organized and documented for this project.

## Scenarios

Use this document when you:
1. start working on a new feature
2. fix a bug in the project

## Task

### Representation

Each task should live in its own folder and use a structured name.

Naming rule: `<ID>-<slug>`, for example
`260407T1154-add-acp-registry`.

ID rule: a time-based identifier in the format `<YYMMDDTHHMM>`, for
example `260407T1154`.

### Root Document

Every task folder must contain a root document named `TASK.md`.

### Template 

You can find them under [template](./template) folder.

- [000000T0000-task-template](./template/000000T0000-task-template)
